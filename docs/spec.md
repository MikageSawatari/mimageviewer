# mimageviewer 仕様書

## 1. アプリケーション概要

Windows 11向けネイティブ画像ビューワー。
ViX（32bit旧来アプリ）の使い勝手を継承しつつ、Rustによるメモリ安全性とGPUアクセラレーションを実現する。

---

## 2. ウィンドウ構成

アプリケーションウィンドウは **1つ**。フルスクリーン表示時のみ別ウィンドウ（ボーダーレス）を生成する。

```
┌─────────────────────────────────────────────────────────┐
│ メニューバー: ファイル | お気に入り | 設定               │
├─────────────────────────────────────────────────────────┤
│ アドレスバー: [フォルダパス テキストボックス         ]  │
├──────────────┬──────────────────────────────────────────┤
│              │                                          │
│  お気に入り  │   サムネイルグリッド (仮想スクロール)    │
│  フォルダ    │                                          │
│  一覧        │                                          │
│              │                                          │
│              │                                          │
└──────────────┴──────────────────────────────────────────┘
```

### 2.1 メニューバー

| メニュー | 項目 |
|---------|------|
| ファイル | 終了 |
| お気に入り | 現在のフォルダを登録 / [登録済みフォルダ一覧] |
| 設定 | サムネイル列数（DragValue） |

### 2.2 アドレスバー

- テキストボックスにフォルダのフルパスを入力
- **Enter** キーでそのフォルダを開く
- フォルダ移動時に自動更新される

### 2.3 お気に入りパネル（左サイドバー）

- 登録されたフォルダ名（またはパス）を縦にリスト表示
- クリックでそのフォルダを開く
- 削除は右クリックメニュー →「削除」

### 2.4 サムネイルグリッド

- 設定で指定した **列数×行数** の固定グリッド
- 各セルはアスペクト比を維持してサムネイルを表示（余白はセル背景色で埋める）
- セルサイズ = ウィンドウの残余領域 ÷ 列数
- **仮想スクロール実装**（下記詳細参照）

## 3. サムネイルグリッド詳細

### 3.1 仮想スクロール

```
総スクロール高さ = ceil(ファイル数 / 列数) × セル高さ

描画対象行 = 現在スクロール位置から見えている行 ± バッファ2行
それ以外の行 = 高さだけ確保、GPUテクスチャ未割り当て
```

- スクロールオフセットは App 自身で管理（egui の自動スクロールは使わない）
- オフセットは常に `cell_size` の整数倍にスナップ（行単位）
- 最大オフセットも行境界にスナップ → 最終行の下に余白が生じる
- マウスホイールは egui の MouseWheel イベントを消費して1行単位で移動

### 3.2 並列サムネイルロード手順

1. フォルダを開く → ファイル一覧を取得（瞬時）
2. グリッドレイアウト・スクロール高さを計算 → 空枠を即座に描画
3. **フォルダごとに専用 rayon スレッドプールを新規作成**（旧フォルダのタスクとプール競合しない）
4. キャンセルトークン（`Arc<AtomicBool>`）を旧タスクへ送信して中断
5. **フェーズ1**：可視範囲の画像を優先して並列ロード
6. **フェーズ2**：残りの画像を並列ロード（キャンセル確認しながら）
7. 完成したサムネイルを `mpsc::channel` でメインスレッドへ送信
8. メインスレッド: `Pending` なサムネイルがある間は毎フレーム `request_repaint()` を呼び、受け取り次第 GPU テクスチャ登録

### 3.3 アップスケール（サムネイル時）

- `image` クレートの `thumbnail()` メソッド（Lanczos3）
- サムネイルサイズ = `max(前フレームのセル幅, 512)` ピクセル、上限 1200px
- 4K・4列環境では概ね 850〜900px のサムネイルを生成

## 4. フルスクリーン表示（未実装 Phase 2）

### 4.1 ウィンドウ仕様

- **ボーダーレスウィンドウ**（タイトルバー・枠なし）
- モニター解像度に合わせたサイズ
- メインウィンドウとは別ウィンドウとして生成

### 4.2 アップスケール戦略

| 状況 | 手法 |
|------|------|
| サムネイル生成 | image::thumbnail()（Lanczos3・CPU） |
| フルスクリーン・画像 < モニター解像度 | NVIDIA NGX DLISR（GPU・AI）※未実装 |
| フルスクリーン・画像 > モニター解像度 | バイキュービック縮小 |

## 5. グリッド表示対象

サムネイルグリッドには以下の2種類のみ表示する。それ以外のファイルは無視する。

| 種別 | 表示内容 |
|------|---------|
| **フォルダ** | フォルダアイコン（📁）＋フォルダ名 |
| **画像ファイル** | サムネイル画像 |

### 5.1 対応画像フォーマット

| フォーマット | 拡張子 |
|------------|--------|
| JPEG | .jpg, .jpeg |
| PNG | .png |
| WebP | .webp |
| BMP | .bmp |

### 5.2 グリッド内のアイテム順

フォルダを先頭に、続いて画像ファイルを表示する（それぞれ名前順・大文字小文字無視）。

## 6. フォルダツリー走査（Ctrl+↑↓）

フォルダツリーを**深さ優先（前順）**でたどる。

```
例：
  c:\folder-1\a
  c:\folder-1\b
  c:\folder-1\b\test1
  c:\folder-1\b\test2
  c:\folder-1\c

c:\folder-1\b を表示中に Ctrl+↓ → c:\folder-1\b\test1 へ
c:\folder-1\b\test2 を表示中に Ctrl+↓ → c:\folder-1\c へ
c:\folder-1\a を表示中に Ctrl+↑ → c:\folder-1 へ（最初の子なので親）
```

- **Ctrl+↓（次）**：子フォルダがあれば最初の子へ、なければ次の兄弟へ、なければ祖先の次の兄弟を再帰検索
- **Ctrl+↑（前）**：前の兄弟の最後の子孫へ、最初の子なら親へ

## 7. キーボード操作

### 7.1 サムネイル表示モード

| キー | 動作 |
|------|------|
| ←↑→↓ | 選択サムネイルを移動（選択セルが見えるよう自動スクロール） |
| Enter | 選択アイテムを開く（フォルダ→移動、画像→フルスクリーン※未実装） |
| BS | 親フォルダへ移動 |
| Ctrl + ↑ | フォルダツリーで前のフォルダへ |
| Ctrl + ↓ | フォルダツリーで次のフォルダへ |

### 7.2 フルスクリーン表示モード（未実装 Phase 2）

| キー | 動作 |
|------|------|
| ← / → | 前/次の画像へ（フォルダはスキップ） |
| ↑ / ↓ | 前/次の画像へ（同上） |
| Ctrl + ↑ | フォルダツリーで前のフォルダへ（サムネイルモードに戻る） |
| Ctrl + ↓ | フォルダツリーで次のフォルダへ（サムネイルモードに戻る） |
| Escape | サムネイルモードに戻る |

## 8. 設定項目

設定は JSON ファイルとして保存（`%APPDATA%\mimageviewer\settings.json`）

| 設定名 | 型 | デフォルト | 説明 |
|--------|-----|---------|------|
| `grid_cols` | u32 | 4 | サムネイルグリッド列数 |
| `grid_rows` | u32 | 3 | サムネイルグリッド行数（スクロール基準） |
| `favorites` | Vec<String> | [] | お気に入りフォルダパス一覧 |

※ 設定の永続化は未実装（Phase 3）

## 10. セキュリティ方針

- 画像デコードは `image` クレート（純粋Rust・メモリ安全）を使用
- WIC は使用しない
- NVIDIA NGX 呼び出し部分のみ `unsafe` ブロックに局所化
