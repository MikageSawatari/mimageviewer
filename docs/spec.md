# mimageviewer 仕様書

## 1. アプリケーション概要

Windows 11向けネイティブ画像ビューワー。
ViX（32bit旧来アプリ）の使い勝手を継承しつつ、Rustによるメモリ安全性とGPUアクセラレーションを実現する。

---

## 2. ウィンドウ構成

アプリケーションウィンドウは **1つ**。フルスクリーン表示時のみ別ウィンドウ（ボーダーレス）を生成する。

```
┌─────────────────────────────────────────────────────────┐
│ メニューバー: ファイル | お気に入り | 設定               │
├─────────────────────────────────────────────────────────┤
│ アドレスバー: [フォルダパス テキストボックス         ]  │
├─────────────────────────────────────────────────────────┤
│ ツールバー: [列数] [縦横比] [ソート] [お気に入り]      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   サムネイルグリッド (仮想スクロール)                   │
│                                                         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 2.1 メニューバー

| メニュー | 項目 |
|---------|------|
| ファイル | フォルダを開く… (Ctrl+O) / メタデータ検索… (Ctrl+F) / 終了 |
| お気に入り | このフォルダを追加… / 編集… / [登録済みフォルダ一覧] |
| 設定 | サムネイル列数 / 比率 / ソート順 / キャッシュ管理… / サムネイル画質… / 統計… / 回転情報をリセット… / 環境設定… |

### 2.2 アドレスバー

- テキストボックスにフォルダのフルパスを入力
- **Enter** キーでそのフォルダを開く
- フォルダ移動時に自動更新される

### 2.3 ツールバー

- 列数（1〜10）、縦横比（7種）、ソート順（4種）、お気に入りフォルダのボタンを配置
- 各セクション・個別項目の表示/非表示をカスタマイズ可能
- すべて非表示にするとツールバー自体が消える

### 2.4 サムネイルグリッド

- 設定で指定した **列数** の可変グリッド（行数は画面に応じて自動）
- 各セルはアスペクト比を維持してサムネイルを表示（余白はセル背景色で埋める）
- セルサイズ = ウィンドウの残余領域 ÷ 列数
- **仮想スクロール実装**（下記詳細参照）

## 3. サムネイルグリッド詳細

### 3.1 仮想スクロール

```
総スクロール高さ = ceil(ファイル数 / 列数) × セル高さ

描画対象行 = 現在スクロール位置から見えている行 ± バッファ2行
それ以外の行 = 高さだけ確保、GPUテクスチャ未割り当て
```

- スクロールオフセットは App 自身で管理（egui の自動スクロールは使わない）
- オフセットは常に `cell_size` の整数倍にスナップ（行単位）
- 最大オフセットも行境界にスナップ → 最終行の下に余白が生じる
- マウスホイールは egui の MouseWheel イベントを消費して1行単位で移動

### 3.2 並列サムネイルロード手順

1. フォルダを開く → ファイル一覧を取得（瞬時）
2. グリッドレイアウト・スクロール高さを計算 → 空枠を即座に描画
3. **フォルダごとに専用 rayon スレッドプールを新規作成**（旧フォルダのタスクとプール競合しない）
4. キャンセルトークン（`Arc<AtomicBool>`）を旧タスクへ送信して中断
5. **フェーズ1**：可視範囲の画像を優先して並列ロード
6. **フェーズ2**：残りの画像を並列ロード（キャンセル確認しながら）
7. 完成したサムネイルを `mpsc::channel` でメインスレッドへ送信
8. メインスレッド: `Pending` なサムネイルがある間は毎フレーム `request_repaint()` を呼び、受け取り次第 GPU テクスチャ登録

### 3.3 サムネイル生成

- 表示用: セルの実ピクセルサイズにリサイズした ColorImage を直接 GPU へ送信（高画質）
- キャッシュ用: 長辺 512px にリサイズ → WebP (lossy, q=75) でエンコード → SQLite に保存
- アイドル時にキャッシュ由来のサムネイルを元画像から再デコードして高画質化（設定で ON/OFF）

### 3.4 ページ単位先読みと Eviction

- `thumb_prev_pages`（デフォルト 2）/ `thumb_next_pages`（デフォルト 4）ページ分のサムネイルを GPU メモリに保持
- 範囲外のサムネイルは `Evicted` 状態にして TextureHandle を drop → VRAM 解放
- `thumb_vram_cap_percent`（デフォルト 50%）で GPU メモリ使用量の安全ネット

## 4. フルスクリーン表示 ✅

### 4.1 ウィンドウ仕様

- **ボーダーレスウィンドウ**（タイトルバー・枠なし）
- モニター解像度に合わせたサイズ
- メインウィンドウとは別ウィンドウとして生成

### 4.2 表示仕様

- 画像をモニター解像度に合わせて表示（バイキュービック縮小/拡大）
- 前後の画像をプリフェッチ（`prefetch_forward` / `prefetch_back` で枚数設定可能）
- マウスを画面上部に移動でファイル情報ヘッダーバーを表示（操作ボタン付き: ℹ / ↶ / ↷ / ▶ / ✕）
- 画面右 1/4 にマウスを移動でメタデータパネルをホバー表示（ピン留め可能）
- メタデータパネル: AI 生成プロンプト (A1111/ComfyUI) + EXIF 撮影情報を表示
- P キー・Space キー・▶ ボタンでスライドショー（間隔設定可能、フォルダ内ループ）
- R / L キーまたは回転ボタンで非破壊回転（SQLite に保存）
- アニメーション GIF・APNG はフルスクリーンでそのまま再生
- 動画は Enter キーで外部プレイヤーを起動
- ビューポートは起動時から非表示で維持し、表示時のちらつきを防止

## 5. グリッド表示対象

サムネイルグリッドには以下を表示する。それ以外のファイルは無視する。

| 種別 | 表示内容 |
|------|---------|
| **フォルダ** | 代表画像サムネイル＋フォルダ名（画像がなければフォルダアイコン📁）。代表画像はフォルダ内を再帰的に探索（最大3階層）し、最初に見つかった画像をソート順で選択 |
| **画像ファイル** | サムネイル画像 |
| **動画ファイル** | サムネイル画像（Windows Shell API 経由） |
| **ZIP ファイル** | 1枚目の画像サムネイル＋ZIP バッジ（ダブルクリックで中を開く） |
| **PDF ファイル** | 1ページ目のサムネイル＋PDF バッジ（ダブルクリックで中を開く） |

### 5.1 対応画像フォーマット

#### 内蔵デコーダー（image クレート）

| フォーマット | 拡張子 |
|------------|--------|
| JPEG | .jpg, .jpeg |
| PNG | .png |
| GIF | .gif |
| WebP | .webp |
| BMP | .bmp |

#### WIC 経由（Windows Imaging Component）

| フォーマット | 拡張子 | 必要なコーデック |
|------------|--------|----------------|
| HEIC / HEIF | .heic, .heif | HEIF 画像拡張機能 |
| AVIF | .avif | AV1 Video Extension |
| JPEG XL | .jxl | JPEG XL Image Extension |
| TIFF | .tiff, .tif | 標準搭載 |
| RAW（各社カメラ） | .dng, .cr2, .cr3, .nef, .nrw, .arw, .srf, .sr2, .raf, .orf, .rw2, .pef, .ptx, .rwl, .iiq | Raw Image Extension |

#### 動画（サムネイル表示のみ）

MP4, AVI, MOV, MKV, WMV, MPG, MPEG

### 5.2 自動除外

- macOS / iPhone の AppleDouble メタデータファイル（`._*`）は自動的に除外

### 5.3 グリッド内のアイテム順

フォルダを先頭に、続いて画像・動画ファイルを表示する。
ソート順は設定で選択可能: ファイル名順 / 数値順（自然順）/ 更新日昇順 / 更新日降順

## 6. フォルダツリー走査（Ctrl+↑↓）

フォルダツリーを**深さ優先（前順）**でたどる。

```
例：
  c:\folder-1\a
  c:\folder-1\b
  c:\folder-1\b\test1
  c:\folder-1\b\test2
  c:\folder-1\c

c:\folder-1\b を表示中に Ctrl+↓ → c:\folder-1\b\test1 へ
c:\folder-1\b\test2 を表示中に Ctrl+↓ → c:\folder-1\c へ
c:\folder-1\a を表示中に Ctrl+↑ → c:\folder-1 へ（最初の子なので親）
```

- **Ctrl+↓（次）**：子フォルダがあれば最初の子へ、なければ次の兄弟へ、なければ祖先の次の兄弟を再帰検索
- **Ctrl+↑（前）**：前の兄弟の最後の子孫へ、最初の子なら親へ
- 画像のないフォルダは `folder_skip_limit`（デフォルト 3）回まで自動スキップ

## 8. 設定項目

設定は JSON ファイルとして `%APPDATA%\mimageviewer\settings.json` に自動保存される。

### 8.1 主要設定

| 設定名 | 型 | デフォルト | 説明 |
|--------|-----|---------|------|
| `grid_cols` | usize | 4 | サムネイルグリッド列数（1〜10） |
| `thumb_aspect` | ThumbAspect | Square | サムネイル縦横比（16:9 / 3:2 / 4:3 / 1:1 / 3:4 / 2:3 / 9:16） |
| `sort_order` | SortOrder | FileName | ソート順（FileName / Natural / MtimeAsc / MtimeDesc） |
| `favorites` | Vec\<FavoriteEntry\> | [] | お気に入りフォルダ（名前付き） |
| `last_folder` | Option\<PathBuf\> | None | 前回開いていたフォルダ |

### 8.2 キャッシュ設定

| 設定名 | 型 | デフォルト | 説明 |
|--------|-----|---------|------|
| `cache_policy` | CachePolicy | Auto | キャッシュ生成ポリシー（Off / Auto / Always） |
| `cache_threshold_ms` | u32 | 25 | Auto モードのしきい値（ms） |
| `cache_size_threshold_bytes` | u64 | 2,000,000 | Auto モードのファイルサイズしきい値 |
| `cache_videos_always` | bool | true | 動画は常にキャッシュ |
| `cache_webp_always` | bool | true | 既存 .webp は常にキャッシュ |
| `cache_pdf_always` | bool | true | PDF ページは常にキャッシュ |
| `cache_zip_always` | bool | true | ZIP 内画像は常にキャッシュ |
| `batch_cache_zip_contents` | bool | false | 一括キャッシュ作成で ZIP 内の全画像をキャッシュ |
| `batch_cache_pdf_contents` | bool | false | 一括キャッシュ作成で PDF 内の全ページをキャッシュ |
| `thumb_px` | u32 | 512 | キャッシュサムネイルの長辺サイズ |
| `thumb_quality` | u8 | 75 | キャッシュの WebP 品質 |

### 8.3 メモリ・先読み設定

| 設定名 | 型 | デフォルト | 説明 |
|--------|-----|---------|------|
| `thumb_prev_pages` | u32 | 2 | サムネイル先読み（前方ページ数） |
| `thumb_next_pages` | u32 | 4 | サムネイル先読み（後方ページ数） |
| `thumb_vram_cap_percent` | u32 | 50 | GPU メモリ上限（VRAM の %） |
| `thumb_idle_upgrade` | bool | true | アイドル時にキャッシュ由来サムネイルを高画質化 |
| `prefetch_back` | usize | 4 | フルスクリーン先読み（前方枚数） |
| `prefetch_forward` | usize | 12 | フルスクリーン先読み（後方枚数） |

### 8.4 その他

| 設定名 | 型 | デフォルト | 説明 |
|--------|-----|---------|------|
| `parallelism` | Parallelism | Auto | 並列読み込みスレッド数 |
| `folder_skip_limit` | usize | 3 | Ctrl+↑↓ で空フォルダをスキップする上限 |
| `folder_thumb_depth` | u32 | 3 | フォルダ代表画像の探索最大階層数（0 で直接の子のみ） |
| `show_toolbar_favorites` | bool | true | ツールバーにお気に入りを表示 |
| `show_toolbar_folder` | bool | true | ツールバーにフォルダ情報を表示 |
| `window_pos` / `window_size` | Option | None | ウィンドウ位置・サイズ（自動保存） |

### 8.5 メタデータ・検索設定（v0.3 追加）

| 設定名 | 型 | デフォルト | 説明 |
|--------|-----|---------|------|
| `exif_hidden_tags` | Vec\<String\> | [MakerNote, UserComment, ...] | EXIF 表示で非表示にするタグ名リスト |
| `slideshow_interval_secs` | f32 | 3.0 | スライドショーの切り替え間隔（秒） |
| `skip_zip_if_folder_exists` | bool | true | 同名フォルダがある ZIP をグリッドから非表示にする（フォルダツリー走査からも除外） |
| `skip_image_if_video_exists` | bool | true | 同名動画がある画像をスキップ（画像は動画サムネイルとして使用） |
| `skip_duplicate_images` | bool | true | 同名で複数拡張子がある画像を優先度でフィルタ |
| `image_ext_priority` | Vec\<String\> | [png, bmp, gif, ...] | 画像拡張子の優先度リスト（先頭が最優先） |

### 8.6 回転情報

回転データは `%APPDATA%\mimageviewer\rotation.db`（SQLite）に保存。元ファイルは変更しない（非破壊）。

## 10. AI 機能詳細

### 10.1 AI モデル一覧

| モデル名 | 用途 | アーキテクチャ | サイズ | ライセンス |
|---------|------|--------------|--------|-----------|
| realesrgan-x4plus | アップスケール（写真向け） | Real-ESRGAN | — | BSD-3-Clause |
| realesrgan-x4plus-anime | アップスケール（アニメ・イラスト向け） | Real-ESRGAN | — | BSD-3-Clause |
| realesr-general-x4v3 | アップスケール（汎用） | Real-ESRGAN | — | BSD-3-Clause |
| realcugan-x4-conservative | アップスケール（漫画向け） | Real-CUGAN | — | GPL-3.0 |
| anime_classification | 画像タイプ分類 | MobileNetV3 | — | MIT |
| mi-gan-512 | 見開き AI 補完 | MI-GAN | — | MIT |
| 1xDeJPG_realplksr_otf | JPEG ノイズ除去（高品質） | RealPLKSR | ~28MB | CC-BY-4.0 |
| 1xDeJPG_OmniSR | JPEG ノイズ除去（軽量） | OmniSR | ~5.5MB | CC-BY-4.0 |

すべてのモデルは exe に `include_bytes!` で埋め込まれ、初回起動時に `%APPDATA%/mimageviewer/models/` に展開される（サイズ一致ならスキップ）。

### 10.2 JPEG ノイズ除去

JPEG 圧縮で発生するブロックノイズ・モスキートノイズを AI で除去する機能。

- **対象**: フルスクリーン表示中の画像（補正パネルから選択）
- **選択肢**: なし / JPEG (RealPLKSR) - 高品質 / JPEG (OmniSR) - 軽量
- **処理順序**: ノイズ除去 → 補正 → アップスケールの順に適用
- **設定**: プリセットごとに `denoise_model` フィールドで保存
- **UI**: 補正パネルの「AI ノイズ除去」ドロップダウンで選択

### 10.3 AI アップスケール

ONNX Runtime + DirectML EP でタイル分割 4x アップスケールを実行。
画像タイプ自動判別（イラスト/漫画/CG/写真）に基づき最適なモデルを自動選択。
補正パネルから手動でモデルを指定することも可能。

### 10.4 見開き AI 補完

見開き表示時の綴じ部分の欠損を MI-GAN で補完。DirectML GPU 推論で ~20ms。

### 10.5 非破壊 AI 修復（消しゴムモード）

フルスクリーンで E キーを押すと消しゴムモードに入り、画像の任意領域をマスクして MI-GAN で補完する。
元画像ファイルは一切変更しない（非破壊）。修復情報はマスクとして DB に保存され、表示時にリアルタイム適用。
タイル分割方式（512×512、64px オーバーラップ、線形ブレンド）で元解像度を維持。

- **ツール**: 囲み（多角形 scan-line fill）、筆（円形、サイズ可変）、縦線、横線
- **描画/消去モード**: マスクの追加・削除を切り替え可能
- **マスク永続化**: SQLite (`mask.db`) に deflate 圧縮した 1bit/pixel マスクを保存。次回表示時に自動適用
- **PDF 対応**: ズーム再レンダリングでサイズが変わった場合、マスクを最近傍法でリスケール
- **復元**: マスク全削除でいつでも元画像に戻せる

## 11. セキュリティ方針

- 画像デコードは `image` クレート（純粋Rust・メモリ安全）を基本とする
- HEIC / AVIF / JXL / TIFF / RAW は Windows の WIC (Windows Imaging Component) 経由でデコード。
  WIC 呼び出しは `unsafe` ブロックに局所化（`src/wic_decoder.rs`）
- NVIDIA NGX 呼び出し部分も `unsafe` ブロックに局所化（Phase 2、未実装）
